GP DATA MANAGER MANUAL

This file explains how the program works today in plain language.

1. WHAT THE PROGRAM DOES

GP Data Manager stores records such as products, drinks, items, or stock lines.
You can:

- add a new item
- edit an existing item
- search items
- back up your data
- restore old backups
- export data to CSV
- track recent numeric changes for an item
- change table layout and keep it for next time


2. HOW TO START IT

Open a terminal in the gp_data folder and run:

python main.py

This opens the desktop window.


3. THE MAIN SCREEN

The window has 2 main parts:

- Left side: the input form
- Right side: the table of saved records

The form is used for adding and editing.
The table is used for browsing, searching, selecting, and inline editing.

The newest saved items are shown first in the table.

The highlighted banner under the form tells you whether you are in New item mode or editing a selected record.
If the form has unsaved edits, that banner changes to show UNSAVED CHANGES.

There is also an Open CSV button for viewing a raw CSV in a separate window.
That CSV viewer is read-only and shows all rows and all columns from the file you opened.
The first time you open a CSV, the app asks whether the file already has a header row.
If you answer No, the viewer creates Column 1, Column 2, Column 3, and so on, and keeps the first row as data.
There is a Last CSV button beside it that reopens the most recently viewed CSV path if that file still exists.
There is also a Recent CSVs button that lists several recently viewed CSV files.

4. ADDING A RECORD

Click New item if you want a blank form.
Fill in the fields on the left.
When you finish the last field, press Enter to save the new record.

Important notes:

- Field 6 is calculated automatically from Field 3 and Field 5 when possible.
- GP, CASH MARGIN, and WITH 70% GP are calculated fields.
- Some text fields are automatically title-cased.
- If another item already has the same Type and Name, the program warns you before adding another one.
- You can still continue and add the duplicate if you really mean to.
- If the app cannot create a safety backup first, it warns you before continuing.


5. EDITING A RECORD

There are 2 ways to edit.

Method A: Main form edit

1. Click a row in the table.
2. The item loads into the main form automatically.
3. Change the values in the form.
4. Click Save changes or press Enter in the last field.

If you want to stop editing that row and enter a completely new item instead, click New item to clear the form and remove the current selection.
Save changes is only active when a record is loaded into the form.
If you changed something and then click another row, click New item, click Delete selected, or close the window, the app asks before throwing those unsaved changes away.
Delete selected is only active when a table row is selected.

Deleting a record:

1. Click a row in the table.
2. Click Delete selected.
3. Confirm the delete.

After a successful delete, the row is removed and the form returns to New item mode.

Method B: Inline edit

1. Double-click a table cell.
2. Type a new value.
3. Press Enter.

This is useful for quick changes.

Important notes about editing:

- Edited rows now stay in the same position in the table instead of jumping to the bottom.
- If an edit would make the item match another existing Type + Name, the program warns you first.
- If you cancel that warning, the edit is not saved.
- If the app cannot create a safety backup before an edit or delete, it asks before continuing.
- If the form has unsaved edits, Delete selected asks whether those edits should be discarded before the delete continues.
- If renamed field labels cannot be saved or applied, the app shows an error and keeps the old labels.
- If an inline edit fails to save, the app shows an error and keeps the inline editor open with your typed value.
- If pressing Enter to submit from the form fails unexpectedly, the app shows an error instead of silently doing nothing.


6. SEARCHING

Use the search box near the top.

- Filtering happens live while you type.
- Search checks Type and Name.
- If there is an exact word match, the app prefers that over a loose substring match.
- Example: searching for gin shows gin items first and avoids matching ginger beer when exact gin matches exist.
- Click the Type column header if you want a small menu showing the saved types.
- Click one type in that menu to show only that type.
- Use Remove type filter at the bottom of that menu to clear the type filter again.
- Click the GP column header if you want a small menu with preset highlight levels.
- You can also pick Custom from that menu and type your own GP threshold.
- Use Clear GP highlight from that same menu to remove the highlighting again.
- That GP highlight choice is saved locally, so it stays there next time you open the app.
- The table only shows matching rows.
- Export uses the rows currently displayed in the table.


7. CHANGE HISTORY

The Change history box is on the form side.

It shows recent numeric changes for the selected item.

Examples of tracked changes:

- price change
- menu price change
- cost/unit change

How to use it:

1. Select an item.
2. Change one of its numeric values.
3. Select the item again if needed.
4. Read the Change history box.

What it shows:

- the selected item name/type
- what field changed
- old value -> new value
- time of change

It now keeps multiple recent numeric changes for the same item.
If there are several entries, use the scrollbar in the box.
Click View full history if you want the whole stored numeric change history for that row.


8. TABLE LAYOUT

You can customize the table.

Reorder columns:

1. Click and hold a column header.
2. Drag it left or right.
3. Release it.

Resize columns:

1. Drag the edge of a column header.

Hide columns:

1. Click Columns.
2. Untick the columns you do not want.
3. Click Apply.

The app remembers:

- column order
- column widths
- hidden/visible columns

So when you close and reopen the program, the layout comes back.


9. BACKUPS

The program creates timestamped backups.

You can also manage backups from the UI:

1. Click Manage backups.
2. Select a backup.
3. Preview it.
4. Restore or delete it.

Manage backups only works with backups created for the storage file you currently have open.

There is no separate Restore backup button on the main window now.
Restore is done from Manage backups.

Important:

- restoring overwrites current storage
- a pre-restore backup is made before restore completes


10. EXPORTING

Click Export CSV.

If the table is filtered by search or by the Type header menu, only the currently shown rows are exported.


11. RAW CSV VIEWER

Use Open CSV if you want to inspect a raw CSV file without importing it into the main app data.

What it does:

- opens the CSV in a separate window
- shows a clear centered processing message while the CSV is opening, while sessions are being combined, and while analysis is being prepared
- uses the same processing-message style across those long CSV actions, and the message closes automatically when that step finishes
- shows all columns from the file
- shows the first preview rows immediately for large files so the window opens faster
- can generate Column 1, Column 2, Column 3, and so on when the file does not have real headers
- lets you type a text search and press Enter when you are ready to run it
- lets you choose which columns stay visible
- shows a small label in the preview telling you whether it is using Headers: Row 1 or Headers: Generated
- lets you click a column header and open a compact popup with a search box and scrollable value list so you can pick one exact value to view, such as one ClassName
- lets you sort text columns A-to-Z or Z-to-A and sort numeric columns low-to-high or high-to-low from that same popup using the real values for that column type
- lets you click Analyze to open a separate analysis window for the current preview result
- lets that analysis window show either a summary table or bar and pie charts for the columns that are still visible in the preview
- lets you save the current preview as a new CSV file based on the current filters, combined rows, and visible columns without changing the original imported file
- can combine session rows like Lunch and Dinner into one row and total their quantity, even when some numeric export columns have names like Textbox73
- keeps the viewer read-only
- lets you scroll across wide files and down long files
- for very large CSV files, only the first block of matching rows is drawn so the window stays usable, and very wide files use a lower live row cap so scrolling and navigation stay smoother

Use Last CSV if you want to reopen the same file again quickly.
Use Recent CSVs if you want to reopen one of several recently viewed files.

Important notes:

- the raw CSV viewer does not import anything into the main product table yet
- the raw CSV viewer search does not run while you are typing; it starts after you press Enter, and then the viewer shows Loading while the search is being applied
- the app remembers the last CSV path and a short recent-files list locally so you do not need to browse to it every time
- the app also remembers whether each CSV path uses a real header row or generated column names, so Last CSV and Recent CSVs reopen the same way
- the app also remembers which preview columns you left visible for each CSV path and restores them by column header when possible if the same file is reordered later
- the app remembers the active preview sort for each CSV path and shows that active sort in the preview summary line
- reopening a CSV through Last CSV or Recent CSVs restores that saved preview state together, so the same header mode, visible columns, and sort come back for that file
- reopening the same unchanged CSV is faster while the app is still open
- reopening the same unchanged CSV after restarting the app is also faster because the viewer can reuse stored preview rows, and some moderate-sized files can also reuse a stored full-row cache for later searches
- repeated searches on larger CSVs can also speed up after the viewer builds a temporary in-memory search index when the file size is still reasonable for your machine
- after the app has fully scanned a large unchanged CSV once, reopening it later is also faster because the viewer can reuse a small metadata sidecar for row counts, encoding, and late columns
- Clear filters resets the preview back to all rows and removes the active search, combine, column filters, and sort, but it does not change which columns you chose to keep visible
- Columns opens a small chooser so you can hide columns you do not want to look at
- Analyze uses only the rows that currently match the preview, so if you search, exact-filter a column, or combine sessions first, the analysis window reflects that current result instead of the original full CSV
- Analyze only includes columns that are currently visible in the preview window
- bar charts in Analyze can show negative values, but pie charts only work with positive values
- bar charts can show all grouped items or just the first or last 5, 10, or 20 items for the chosen visible label and value columns in highest-to-lowest order
- bar charts can be switched between vertical and horizontal layouts
- vertical bar charts scroll horizontally when needed, and horizontal bar charts scroll vertically when needed
- pie charts keep the most important slices visible and group the smaller remainder into Other so the chart stays readable
- Save As CSV creates a new CSV file from what the preview currently shows logically, and the original imported file stays unchanged
- Save As CSV opens in your shared `Favorites\csv_exports` folder by default, and the folder is created automatically if needed
- clicking a column header opens a compact popup with a search field first, a scrollable value list for that column, sort controls for that column type, and a Clear filter action for removing that exact-value filter again
- Combine sessions groups rows that only differ by session and adds their quantity together when the CSV has matching session and quantity columns
- if a CSV is very large, the preview summary tells you when it is showing preview rows first or only the first matching rows to avoid freezing the UI


12. COMMAND LINE

The program also has CLI commands.

Examples:

python main.py list
python main.py backup
python main.py restore
python main.py export output.csv


13. SETTINGS THAT ARE SAVED

The program saves settings in settings.json.
That file is local on your machine and is not meant to be pushed to Git.

This includes:

- field labels
- column order
- column widths
- visible columns
- the GP row highlight threshold
- the last CSV preview path used by Last CSV
- the recent CSV preview paths used by Recent CSVs
- the saved raw CSV preview state for each CSV path, including visible columns, active sort, and remembered header-row choice, with header-aware restore when possible for reordered CSVs

Important:

- if renamed field labels cannot be saved or applied, the app keeps the old labels and shows an error
- if saving column layout or GP highlight settings fails, the app warns that the change may not still be there after restart
- if you try to close the window with unsaved form edits, the app asks before discarding them


14. IF SOMETHING LOOKS WRONG

If the table or form looks odd:

- close the program and open it again
- if the app warns that storage could not be opened fully, close other programs using the file and then reopen it
- check that the item is selected before reading the Change history box
- check the scrollbar in the Change history box if only one entry is visible

If data looks wrong after an edit:

- open Manage backups
- preview recent backups
- restore the needed one

If you still see two items with the same Type and Name:

- that usually means both rows already existed in saved data
- the new warning stops accidental new duplicates, but it does not automatically merge old ones

If the CSV viewer does not reopen the file you expect:

- use Open CSV to pick the file again
- check that the old CSV still exists at the remembered path
- if the file changed on disk, the app reloads it again instead of reusing the old preview
- if the first row is being treated the wrong way, reopen it with Open CSV and answer the header-row question differently


15. CURRENT LIMITATIONS

- The main form shows recent numeric changes; full history is available from View full history.
- At least one table column must stay visible.
- Duplicate warnings are a safeguard, not a forced block. You can still continue if you explicitly choose to.
- GUI tests may be skipped on systems without Tk support, but the app itself still targets Tkinter.


16. QUICK DAILY WORKFLOW

1. Start the app with python main.py
2. Search or select an item
3. Edit inline or use Save changes from the main form
4. Check the Change history box or open View full history
5. Export or manage backups when needed
